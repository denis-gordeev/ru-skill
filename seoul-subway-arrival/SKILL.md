# Прибытие поездов метро Сеула

## Что делает этот навык

Сводит информацию о прибывающих поездах по станциям через Open API реального времени метрополитена Сеульского открытого портала данных.

## Когда использовать

- "Через сколько минут прибудет поезд на Каннам?"
- "Покажи информацию о прибытии на 1-й линии станции Сеул"
- "Сведи поезда, которые скоро прибудут на станцию Чамсиль"

## Предварительные требования

- API-ключ Сеульского открытого портала данных
- опционально: `jq`

## Требуемые переменные окружения

- `SEOUL_OPEN_API_KEY`

### Порядок разрешения учётных данных

1. **Если уже есть в переменных окружения** — использовать как есть.
2. **Если агент использует собственный secret vault (1Password CLI, Bitwarden CLI, macOS Keychain и т.д.)** — извлечь оттуда и внедрить в переменную окружения.
3. **`~/.config/k-skill/secrets.env`** (fallback по умолчанию) — plain dotenv файл, права `0600`.
4. **Если ничего нет** — спросить у пользователя и сохранить в 2 или 3.

Сохранение в путь по умолчанию — лишь fallback, не обязательное требование.

## Входные данные

- Название станции
- Опционально: количество записей для получения

## Рабочий процесс

### 1. Убедиться в наличии учётных данных

Проверить, установлена ли переменная окружения `SEOUL_OPEN_API_KEY`. Если нет — получить её согласно порядку разрешения учётных данных выше.

Не переключаться автоматически на неофициальные зеркальные API или другие источники из-за отсутствия секретов.

### 2. Запросить официальный endpoint прибытия на станцию

Сеульский API метро реального времени предоставляет информацию о прибытии в реальном времени по станциям в формате JSON/XML. Базовый пример запроса использует следующий паттерн.

```bash
curl -s "http://swopenAPI.seoul.go.kr/api/subway/${SEOUL_OPEN_API_KEY}/json/realtimeStationArrival/0/8/강남"
```

### 3. Свести ответ

По возможности свести только следующие поля.

- Линия
- Направление (вверх/вниз или внешняя/внутренняя)
- Первое сообщение о прибытии
- Второе сообщение о прибытии
- Время до прибытия (если есть, в секундах)

### 4. Консервативно относиться к данным реального времени

Данные реального времени могут меняться каждые несколько секунд, поэтому в ответе всегда указывать время запроса.

## Готово, когда

- Сведены поезда, ожидающие прибытия на запрошенную станцию
- Указано время запроса данных реального времени
- Ключ не раскрыт

## Режимы сбоев

- API-ключ не установлен
- Превышение квоты
- Несоответствие названия станции

## Примечания

- Руководство Сеульского открытого портала данных указывает, что на Open API метро реального времени могут быть суточные лимиты вызовов
- Пути endpoint могут изменяться с версией API, поэтому при сбое перепроверять актуальные примеры URL в консоли наборов данных
