## Что делает этот навык

Ищет товары на Яндекс Маркете по запросу и открывает карточку конкретного товара по URL из результатов поиска. Нормализует цену, рейтинг, число оценок и базовые характеристики без API key и без браузерной автоматизации.

## Когда использовать

- Когда нужно быстро найти товар на российском маркетплейсе по названию
- Когда нужен read-only обзор карточки товара с ценой и характеристиками
- Когда нужно заменить legacy-сценарий `daiso-product-search` на российский источник

## Предварительные требования

- Публичный доступ к `market.yandex.ru`
- `npm install yandex-market-search`
- Никакие секреты, API-ключи и прокси не нужны

## Входы

- Поисковый запрос товара, например `iphone 16`, `пылесос xiaomi`, `кофемашина delonghi`
- Необязательная страница результатов `page`
- URL карточки товара из `searchProducts(...)`

## Основной сценарий

1. Вызвать `searchProducts("название товара")`
2. Выбрать релевантный результат по `title`, `price`, `rating` и inline-характеристикам
3. Передать `url` из результата в `getProduct(url)` для чтения полной карточки
4. Вернуть нормализованные поля: цена, рейтинг, число оценок, описание и характеристики

## Готово, когда

- В поиске есть нормализованный список карточек с canonical URL без tracking query
- Карточка товара возвращает `title`, `brand`, `price`, `rating`, `reviewCount` и `specs`
- Источник явно помечен как `market.yandex.ru`

## Режимы отказа

- Яндекс Маркет вернул не-200 статус — пробрасывается ошибка с кодом ответа
- Вёрстка изменилась и ключевые поля не распознаны — частичные поля остаются `null`, но канонический URL и доступные характеристики возвращаются
- Пустая выдача по запросу — возвращается пустой `results`

## Примечания

- Текущий MVP опирается на серверно отрендеренный HTML поисковой выдачи и product card page
- Write-операции не поддерживаются: корзина, покупка, избранное и отзывы вне scope
- Если Яндекс Маркет переведёт выдачу в чистый CSR или антибот-режим, источник нужно будет переоценить в `docs/sources.md`
