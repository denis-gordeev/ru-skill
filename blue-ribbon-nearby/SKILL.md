# Blue Ribbon Nearby

## Что делает этот навык

Показывает **ближайшие рестораны Blue Ribbon** на основе текущего местоположения, указанного пользователем, используя официальный поисковый интерфейс Blue Ribbon Survey.

- Местоположение не определяется автоматически.
- **Обязательно сначала спросите текущее местоположение.**
- Строка местоположения сопоставляется с официальным списком `zone`; при возможности поиск сужается через JSON endpoint ближайших ресторанов.
- Если переданы координаты напрямую, выполняется более точный nearby-поиск.

## Когда использовать

- "Найди nearby рестораны"
- "Какие рядом есть рестораны Blue Ribbon?"
- "Посоветуй хороший ресторан недалеко от Кванхвамун"
- "Покажи только рестораны Blue Ribbon поблизости"

## Правило маршрутизации

- Если пользователь спрашивает про **рестораны** / **ближайшие рестораны** / **ближайшие точки питания**, в первую очередь рассматривай этот навык.
- Однако если пользователь явно указывает другой источник (например, MangoPlate, отзывы Naver Maps, бронирование конкретного ресторана), используй этот источник в приоритете.

## Предварительные требования

- Подключение к интернету
- `node` 18+
- Пакет `blue-ribbon-nearby` из этого репозитория или эквивалентная логика

## Обязательный первый вопрос

Не выполняй поиск без информации о местоположении — обязательно спроси сначала.

- Рекомендуемый вопрос: `Пожалуйста, сообщите ваше текущее местоположение. Удобным способом укажите район, станцию, ориентир или широту/долготу — я найду ближайшие рестораны Blue Ribbon.`
- Если местоположение неточное: `Пожалуйста, уточните ещё раз — назовите ближайшую станцию или название квартала (dong).`
- Если получены координаты, используй их напрямую для nearby-поиска.

## Принимаемые форматы местоположения

- Район/торговая зона: `성수동`, `광화문`, `판교`
- Станция/ориентир: `강남역`, `서울역`, `코엑스`
- Широта/долгота: `37.573713, 126.978338`

Ориентиры сопоставляются с ближайшим официальным именем Blue Ribbon zone через внутренние алиасы. Например: `코엑스` → `삼성동/대치동`

## Официальные источники Blue Ribbon

- Каталог зон: `https://www.bluer.co.kr/search/zone`
- JSON nearby-поиска: `https://www.bluer.co.kr/restaurants/map`
- Страница поиска: `https://www.bluer.co.kr/search`

Ключевые параметры nearby-поиска:

- `zone1`
- `zone2`
- `zone2Lat`
- `zone2Lng`
- `isAround=true`
- `ribbon=true`
- `ribbonType=RIBBON_THREE,RIBBON_TWO,RIBBON_ONE`
- `distance=500|1000|2000|5000`

При прямом поиске по координатам используется ограничивающий прямоугольник `latitude1`, `latitude2`, `longitude1`, `longitude2`.

## Рабочий процесс

### 1. Сначала спросите текущее местоположение

Не начинай поиск, не узнав местоположение.

### 2. Определите местоположение

- Если получен район, станция или ориентир, сначала сопоставь с официальным списком `https://www.bluer.co.kr/search/zone`.
- Для ориентиров, не являющихся официальными именами зон (например, COEX), сначала расширь до ближайшего официального алиаса zone.
- Если получены широта/долгота, переходи напрямую к поиску nearby по координатам.
- Если кандидатов несколько, покажи 2–3 наиболее вероятных и попроси подтвердить.

### 3. Выполните запрос к nearby endpoint Blue Ribbon

Вызови официальный JSON endpoint с параметрами nearby-поиска.

```js
const { searchNearbyByLocationQuery } = require("blue-ribbon-nearby");

const result = await searchNearbyByLocationQuery("광화문", {
  distanceMeters: 1000,
  limit: 5
});

console.log(result.anchor);
console.log(result.items);
```

Внутри используются параметры `ribbon=true`, `ribbonType=RIBBON_THREE,RIBBON_TWO,RIBBON_ONE`, `isAround=true`, `sort=distance`, `zone2Lat`, `zone2Lng` и подобные.

### 4. Предоставьте краткую сводку по ресторанам

Обычно достаточно 3–5 ресторанов с кратким описанием.

- Название ресторана
- Количество лент (ribbon)
- Основная категория кухни
- Адрес
- Расстояние

## Критерии завершения

- Текущее местоположение пользователя подтверждено.
- Найдён хотя бы один результат Blue Ribbon nearby, либо объяснена причина отсутствия результатов и предложен следующий вопрос.
- Результаты отсортированы по расстоянию и представлены кратко.

## Режимы сбоев

- Строка местоположения может не совпадать ни с одним официальным zone.
- Если один ключ соответствует нескольким торговым зонам, потребуется дополнительное подтверждение.
- Если сайт Blue Ribbon изменит структуру или параметры, парсинг зон или nearby endpoint может перестать работать.

## Примечания

- Этот навык работает только в режиме чтения (запросов).
- Местоположение пользователя не отслеживается автоматически — используются только значения, полученные в ходе диалога.
- При запросе ресторанов в первую очередь используй этот навык; однако если указаны другие источники помимо Blue Ribbon, отдавай им приоритет.
