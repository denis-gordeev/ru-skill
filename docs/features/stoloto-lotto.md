# Лотереи Столото

Read-only-навык для получения публичных результатов тиражей Столото через архивные страницы.

## Что умеет

- Получать последние тиражи для популярных лотерей: `4 из 20`, `5 из 36`, `6 из 45`, `7 из 49`, `Русское лото`, `Топ-3` и других.
- Возвращать номер тиража, дату, выпавшие числа и размер суперприза.
- Работать без авторизации, секретов и прокси.

## Что не умеет

- Проверять конкретные билеты по номеру.
- Оформлять заявки на выплату выигрыша.
- Работать с личным кабинетом Столото.

## Источник данных

- Архив тиражей: `https://www.stoloto.ru/{game}/archive`
- Столото — официальный оператор государственных лотерей в России.
- Данные извлекаются из HTML-страниц архива через regex-парсинг, аналогично `postcalc-postcodes`.

## Поддерживаемые лотереи

| Slug | Название | Чисел в комбинации |
| --- | --- | --- |
| `4x20` | Спортлото 4 из 20 | 8 (4 + 4) |
| `5x36` | Спортлото 5 из 36 | 5 (+ 1) |
| `6x45` | Спортлото 6 из 45 | 6 |
| `7x49` | Спортлото 7 из 49 | 7 |
| `12x24` | Спортлото 12 из 24 | 12 |
| `ruslotto` | Русское лото | varies |
| `top3` | Топ-3 | 3 |
| `5x2` | 5 из 2 | varies |

## Как использовать через package

```js
const { getArchiveDraws, SUPPORTED_GAMES } = require("stoloto-lotto");

console.log(SUPPORTED_GAMES);
// ["4x20", "5x36", "6x45", "7x49", "12x24", "ruslotto", "top3", "5x2"]

const draws = await getArchiveDraws("6x45");

console.log(draws.gameName);
// "Архив тиражей лотереи «Спортлото 6 из 45»"

console.log(draws.draws[0]);
// {
//   drawNumber: 98765,
//   date: "07.04.2026",
//   numbers: [15, 24, 33, 41, 7, 45],
//   prize: "15 000 000"
// }
```

## Примеры запросов для агента

### Проверка последних результатов 6 из 45

```
Проверь последние тиражи лотереи «6 из 45» на Столото.
```

Агент вызовет `getArchiveDraws("6x45")` и покажет последние выпавшие числа и размер суперприза.

### Сравнение двух лотерей

```
Какие числа выпали в последних тиражах 4 из 20 и 6 из 45?
```

Агент вызовет `getArchiveDraws("4x20")` и `getArchiveDraws("6x45")` и сравнит результаты.

## Ограничения

- Столото не предоставляет публичного REST API; данные извлекаются из HTML-страниц.
- Если вёрстка страниц изменится, парсинг может сломаться — в этом случае обновите fixture-тесты и парсер.
- Размер приза возвращается как строка, так как форматирование может включать пробелы и единицы измерения.

## Тесты

```bash
npm test --workspace stoloto-lotto
```

Тесты используют fixture-based подход с сохранёнными HTML-страницами архива, чтобы CI не зависел от живой вёрстки stoloto.ru.
