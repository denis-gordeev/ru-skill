# Гайд по CLI для KakaoTalk на macOS

## Что умеет этот сценарий

- Читать список недавних чатов в KakaoTalk на macOS.
- Показывать последние сообщения из конкретной комнаты.
- Искать по всем чатам по ключевому слову.
- Безопасно отправлять тестовые сообщения в чат с собой.
- После подтверждения пользователя отправлять сообщение в выбранный чат.

## Что нужно заранее

- macOS
- Установленный KakaoTalk for Mac
- Homebrew
- `brew install silver-flight-group/tap/kakaocli`
- Для терминала выданы права **Full Disk Access** и **Accessibility**

Если приложение KakaoTalk ещё не установлено, его можно поставить через `mas`.

```bash
brew install mas
mas account
mas install 869223134
```

## Входные данные

- Название чата
- Ключевое слово для поиска
- Период (`--since 1h`, `--since 7d` и т.п.)
- Текст сообщения
- Режим теста (`--me`, `--dry-run`)

## Базовый поток

1. Проверить, что установлен KakaoTalk for Mac и `kakaocli`.
2. Сначала убедиться через `kakaocli status` и `kakaocli auth`, что права и доступ к БД работают.
3. Для чтения и поиска запускать команды в JSON-режиме, затем уже сводить ответ в человекочитаемый вид.
4. Перед реальной отправкой сначала тестировать через `--me` или `--dry-run`.
5. В чужие чаты отправлять сообщения только после явного подтверждения пользователя.

## Примеры

```bash
kakaocli status
kakaocli auth
kakaocli chats --limit 10 --json
kakaocli messages --chat "지수" --since 1d --json
kakaocli search "회의" --json
kakaocli send --me _ "테스트 메시지"
kakaocli send --dry-run "팀 공지방" "오늘 3시에 만나요"
```

## Ограничения

- Без **Full Disk Access** могут падать даже read-only команды.
- Без **Accessibility** ломается отправка и автоматизация harvest-подобных действий.
- Это сценарий только для macOS; не нужно молча переключаться на Windows/Linux-обходы.
- Сообщения другим людям нельзя отправлять без отдельного подтверждения.
