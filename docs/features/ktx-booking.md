# Гайд по бронированию KTX

## Что умеет этот сценарий

- Искать поезда KTX/Korail.
- Проверять наличие мест.
- Оформлять бронь.
- Показывать текущие бронирования.
- Отменять бронь.

## Что нужно заранее

- Python 3.10+
- `python3 -m pip install korail2 pycryptodome`
- Прочитать [общую настройку](../setup.md)
- Прочитать [политику по секретам](../security-and-secrets.md)

## Нужные переменные окружения

- `KSKILL_KTX_ID`
- `KSKILL_KTX_PASSWORD`

### Порядок разрешения учётных данных

1. Если переменные уже заданы в окружении, использовать их.
2. Если агент работает с отдельным secret vault, можно взять значения оттуда и инжектировать как env.
3. Если env нет, сначала искать их в `~/.config/ru-skill/secrets.env`, затем в legacy fallback `~/.config/k-skill/secrets.env`.
4. Если источников нет, запросить данные у пользователя и сохранить их в vault или `secrets.env`.

## Входные данные

- Станция отправления
- Станция прибытия
- Дата: `YYYYMMDD`
- Желаемое время старта: `HHMMSS`
- Количество пассажиров и типы пассажиров
- Предпочтение по местам
- `train_id`, скопированный из результата поиска

## Зачем нужен helper из репозитория

Одних примеров из публичного `korail2` 0.4.0 недостаточно: запросы могут упираться в Dynapath anti-bot и падать с `MACRO ERROR`.

`scripts/ktx_booking.py` в этом репозитории дополнительно выставляет:

- `x-dynapath-m-token`
- `Sid`
- актуальную версию приложения `250601002`
- свежий Android user-agent

## Базовый поток

1. Если нет `korail2` или `pycryptodome`, сначала установить зависимости.
2. Если `KSKILL_KTX_ID` и `KSKILL_KTX_PASSWORD` не заданы, получить их по порядку разрешения выше.
3. Сначала выполнить поиск поездов через helper.
4. Показать `index`, `train_id`, время отправления и прибытия, тип поезда и наличие мест.
5. Бронировать только после того, как пользователь явно выбрал конкретный поезд.
6. Для просмотра и отмены брони сначала заново идентифицировать нужное бронирование.

## Примеры

Поиск:

```bash
python3 scripts/ktx_booking.py search 서울 부산 20260328 090000 --limit 5
```

Чтобы видеть и поезда без мест, добавьте `--include-no-seats`. Чтобы включить поезда с waitlist, добавьте `--include-waiting-list`.

`train_id` из JSON-ответа служит стабильным селектором конкретного поезда на момент поиска. Для бронирования его нужно передавать без изменений. Если поезд уже ушёл из выдачи, helper завершится ошибкой и заставит выполнить новый поиск.

Бронирование:

```bash
python3 scripts/ktx_booking.py reserve 서울 부산 20260328 090000 --train-id <train_id> --seat-option general-first
```

Если при отсутствии мест нужно попробовать waitlist, включите `--include-waiting-list` на этапе поиска и `--try-waiting` на этапе бронирования.

Просмотр бронирований:

```bash
python3 scripts/ktx_booking.py reservations
```

Отмена:

```bash
python3 scripts/ktx_booking.py cancel <reservation_id>
```

Все ответы приходят в JSON и содержат номер брони, дедлайн выкупа и стоимость. Автоматизация заканчивается на бронировании и не закрывает оплату. Историческая формулировка из legacy-доков: `결제는 제외`.

## Ограничения

- Этот сценарий не следует смешивать с SRT: это отдельная поверхность и отдельные учётные данные.
- Секреты нужно подавать через env, vault или поддерживаемые `secrets.env` файлы.
- Полная автоматизация оплаты не входит в scope.
- Если Korail снова изменит anti-bot правила, helper придётся актуализировать.
