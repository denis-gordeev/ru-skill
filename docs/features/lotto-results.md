# Гайд по результатам корейской лотереи

## Что умеет этот сценарий

- Показывать последний доступный тираж.
- Возвращать выигрышные номера для конкретного тиража.
- Сравнивать пользовательские номера с выпавшими.
- Показывать распределение призов.

## Что нужно заранее

- Node.js 18+
- После публикации: `npm install -g k-lotto`
- Перед запуском: `export NODE_PATH="$(npm root -g)"`
- Для локальной разработки из этого репозитория: `npm install` в корне

## Входные данные

- Номер тиража или `latest`
- Необязательно: шесть пользовательских номеров

## Базовый поток

1. Если пакет не установлен, сначала установить его, а не искать обходы.
2. Определить, нужен последний тираж или конкретный номер.
3. Свести выигрышные номера, дату розыгрыша и структуру призов.
4. Если пользователь передал свои номера, посчитать совпадения и категорию выигрыша.

## Пример

```bash
NODE_PATH="$(npm root -g)" node - <<'JS'
const lotto = require("k-lotto");
lotto.getDetailResult(1216).then((result) => console.log(JSON.stringify(result, null, 2)));
JS
```

## Ограничения

- Пользовательские номера не стоит хранить дольше одного ответа.
- Последний тираж читается из HTML официальной страницы, а детальные данные берутся из официального JSON-ответа.
