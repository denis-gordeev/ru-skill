# Гайд по обработке HWP-документов

## Что умеет этот сценарий

- Конвертировать `.hwp` в JSON, Markdown и HTML.
- Извлекать изображения из документа.
- Обрабатывать документы пакетно по каталогу.
- На Windows с установленным Hancom Office переходить к прямому управлению документом.

## Что нужно заранее

- Для базового пути: Node.js 18+
- Базовый пакет: `npm install -g @ohah/hwpjs`
- Перед запуском: `export NODE_PATH="$(npm root -g)"`
- Только для прямого управления: Windows, установленный HWP и Python 3.7+

## Какой путь выбирать

### Базовый вариант: `@ohah/hwpjs`

Используйте `@ohah/hwpjs`, если:

- это macOS, Linux или CI
- нужны чтение, конвертация, извлечение изображений или пакетная обработка
- даже на Windows нет уверенности, что локальный HWP установлен и доступен

### Исключение: `hwp-mcp`

Переходите к `hwp-mcp` только если одновременно выполняются все условия:

- это Windows
- Hancom HWP реально установлен
- нужна именно живая автоматизация редактора: создание файла, вставка текста, заполнение таблиц и т.д.

Правило по умолчанию простое: конвертация через `@ohah/hwpjs`, прямое управление через `hwp-mcp`.

## Базовый поток

1. Определить платформу через `node -p "process.platform"`.
2. Если это не `win32`, сразу использовать `@ohah/hwpjs`.
3. Если это `win32`, но требование к прямому управлению не подтверждено, всё равно использовать `@ohah/hwpjs`.
4. Только при подтверждённой потребности в управлении документом и установленном HWP переключаться на `hwp-mcp`.
5. После выполнения проверить факт создания файлов и содержимое результата.

## Примеры

### Конвертация в JSON

```bash
hwpjs to-json document.hwp -o output.json --pretty
```

### Конвертация в Markdown с изображениями

```bash
hwpjs to-markdown document.hwp -o output.md --include-images
```

`--include-images` встраивает изображения в Markdown как base64 `data:` URI, а не пишет их в отдельные файлы.
Строка для doc-регрессии: `Markdown 출력` должен включать `data:` или `base64`.

### Конвертация в HTML

```bash
hwpjs to-html document.hwp -o output.html
```

### Извлечение изображений

```bash
hwpjs extract-images document.hwp -o ./images
```

### Пакетная обработка

```bash
hwpjs batch ./documents -o ./output --format json --recursive
```

## Что проверять в результате

- Для JSON: создан ли файл и как выглядит верхнеуровневая структура.
- Для Markdown: если использовался `--include-images`, отдельные файлы изображений не обязательны; нужно проверять встроенные `data:` URI.
- Для HTML: открывается ли результат в браузере.
- Для extraction: появились ли реальные файлы изображений в выходной директории.
- Для batch: не слишком ли расходится число входных и выходных файлов.

Если изображения нужны отдельными файлами, вместо `--include-images` используйте `--images-dir`.

## Когда нужен прямой контроль

`hwp-mcp` рассматривается только на Windows и только при установленном Hancom HWP.

```bash
git clone https://github.com/jkf87/hwp-mcp.git
cd hwp-mcp
pip install -r requirements.txt
```

После этого можно подключаться к MCP-серверу и выполнять операции вроде создания документа, вставки текста, заполнения таблиц и сохранения.

## Ограничения

- Не стоит пытаться запускать `hwp-mcp` обходным путём на Linux или macOS.
- Если потребность в прямом управлении слабая, `@ohah/hwpjs` надёжнее и проще.
- После batch-задач всегда полезно сравнить число входных и выходных документов.
