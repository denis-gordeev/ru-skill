# MOEX Shares

## Что делает навык

`moex-shares` - это read-only-обёртка над публичным ISS API Московской биржи для акций с основной доски `TQBR`. Навык нормализует тикер, краткие метаданные инструмента и задержанный рыночный снимок.

## Когда использовать

- Нужна последняя доступная публичная цена по тикеру MOEX
- Нужны `ISIN`, `lotSize`, предыдущая цена или валюта расчёта
- Нужно быстро получить страницу доступных тикеров без браузера и без секретов

## Предварительные условия

- Node.js 18+
- После публикации: `npm install -g moex-shares`
- Перед запуском: `export NODE_PATH="$(npm root -g)"`
- При разработке из этого репозитория: `npm install`

## Базовый сценарий

1. Если пакет не установлен, сначала поставить `moex-shares`.
2. Получить сводку по тикеру через `getSecurityOverview`.
3. Если нужен список инструментов, использовать `listShares`.

## Пример: сводка по тикеру

```bash
NODE_PATH="$(npm root -g)" node - <<'JS'
const { getSecurityOverview } = require("moex-shares");

getSecurityOverview("SBER").then((result) => {
  console.log(JSON.stringify(result, null, 2));
});
JS
```

## Пример: первая страница тикеров

```bash
NODE_PATH="$(npm root -g)" node - <<'JS'
const { listShares } = require("moex-shares");

listShares({ start: 0 }).then((result) => {
  console.log(JSON.stringify(result, null, 2));
});
JS
```

## Что возвращать пользователю

- Тикер и короткое название инструмента
- `ISIN`, `lotSize`, валюта расчёта и предыдущая цена
- `lastPrice`, изменение и время обновления, если спрашивали про рынок
- Явную оговорку, что публичный ISS обычно даёт delayed snapshot

## Ограничения

- Навык покрывает только `stock/shares` и по умолчанию доску `TQBR`
- Секреты не нужны, но интернет-доступ обязателен
- Это не торговый и не брокерский интерфейс, а только публичное read-only чтение
