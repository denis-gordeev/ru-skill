# Гайд по прибытию поездов сеульского метро

## Что умеет этот сценарий

- Получать время прибытия поездов по станции в реальном времени.
- Показывать направление движения и тип линии.
- Возвращать первое и второе сообщения о прибытии.

## Что нужно заранее

- Прочитать [общую настройку](../setup.md)
- Прочитать [политику по секретам](../security-and-secrets.md)
- Получить API key Seoul Open Data Plaza

## Нужные переменные окружения

- `SEOUL_OPEN_API_KEY`

### Порядок разрешения учётных данных

1. Если переменная уже есть в окружении, использовать её.
2. Если агент работает через отдельный secret vault, можно подтянуть ключ оттуда.
3. Если env нет, сначала искать `SEOUL_OPEN_API_KEY` в `~/.config/ru-skill/secrets.env`, затем в `~/.config/k-skill/secrets.env`.
4. Если ни одного источника нет, запросить ключ у пользователя и сохранить его в vault или `secrets.env`.

## Входные данные

- Название станции
- Необязательно: число записей в ответе

## Базовый поток

1. Если `SEOUL_OPEN_API_KEY` не задан, получить его по описанному порядку разрешения.
2. Запросить live-данные по станции.
3. Свести линию, направление, сообщения о прибытии и момент обновления.

## Пример

```bash
curl -s "http://swopenAPI.seoul.go.kr/api/subway/${SEOUL_OPEN_API_KEY}/json/realtimeStationArrival/0/8/강남"
```

## Ограничения

- Данные real-time и могут меняться каждые несколько секунд.
- Если написание станции отличается от ожидаемого API, ответ может быть пустым.
- Возможны суточные лимиты и квоты.
