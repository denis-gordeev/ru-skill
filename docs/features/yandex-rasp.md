# Яндекс.Расписания — yandex-rasp

## Обзор

Пакет `yandex-rasp` — это read-only клиент для публичного API Яндекс.Расписаний. Покрывает три базовых сценария: поиск станции по названию, расписание для станции и поиск рейсов между двумя точками.

## Предварительные требования

- Бесплатный API-ключ Яндекс.Расписаний: https://yandex.ru/dev/rasp/
- Переменная окружения `YANDEX_RASP_API_KEY` или передача ключа через `opts.apiKey`

## Входы

| Сценарий | Вход | Пример |
| --- | --- | --- |
| Поиск станции | Подстрока названия | `searchStations("Казанский")` |
| Расписание станции | Код станции + опционально дата, тип, направление | `getStationSchedule("s9600013", { date: "2026-04-10" })` |
| Поиск маршрута | Коды двух станций/городов + опционально дата, тип транспорта | `searchTrips("c146", "c159", { date: "2026-04-10" })` |

## Базовый сценарий

1. Вызвать `searchStations("название")` для поиска нужной станции
2. Взять `yandexCode` из результата
3. Вызвать `getStationSchedule(yandexCode, opts)` для получения расписания или `searchTrips(from, to, opts)` для поиска маршрута

      "from": { "code": "s9600013", "title": "Москва, Казанский вокзал" },

## Что важно проверять в ответе

- `thread.transportType` — тип транспорта: `suburban`, `train`, `bus`, `plane`
- `departure` / `arrival` — ISO 8601 с часовым поясом
- `days` — регулярность рейса
- `ticketsInfo.places` — доступные классы и цены
- `hasTransfers` — наличие пересадок
- `pagination.total` — сколько всего результатов доступно

## Ограничения

- Требуется API-ключ (бесплатный, но регистрация обязательна)
- Справочник станций (~40 МБ) лучше кэшировать на клиенте
- Read-only, без покупки билетов
- Не покрывает полный охват всех типов транспорта, если они не указаны в документации API
