# Гайд по поиску товаров Daiso

## Что умеет этот сценарий

- Искать официальные магазины Daiso по названию.
- Искать товары по названию или общему поисковому запросу.
- Проверять pickup-остатки в конкретном магазине.
- При необходимости добавлять ориентир по онлайн-наличию.

## Что нужно заранее

- Доступ в интернет
- `node` 18+

## Входные данные

- Название магазина
  - Например: `강남역2호점`
  - Например: `스타필드하남점`
- Название товара или поисковый запрос
  - Например: `VT 리들샷 100`
  - Например: `리들샷 300`

## Официальные поверхности

- Поиск магазинов: `https://www.daisomall.co.kr/api/ms/msg/selStr`
- Детали магазина: `https://www.daisomall.co.kr/api/dl/dla-api/selStrInfo`
- Поиск товаров: `https://www.daisomall.co.kr/ssn/search/SearchGoods`
- Сводка по товарам: `https://www.daisomall.co.kr/ssn/search/GoodsMummResult`
- Pickup-остатки магазина: `https://www.daisomall.co.kr/api/pd/pdh/selStrPkupStck`
- Необязательная онлайн-проверка: `https://www.daisomall.co.kr/api/pdo/selOnlStck`

## Базовый поток

1. Если нет названия магазина, сначала запросить его.
2. Если нет товара, уточнить название или поисковую фразу.
3. Через `selStr` найти кандидатов магазинов и при необходимости добрать детали через `selStrInfo`.
4. Через `SearchGoods` найти кандидатов товаров.
5. Через `selStrPkupStck` проверить остаток товара в выбранном магазине.
6. При необходимости сохранить `onldPdNo` из `SearchGoods` и проверить онлайн-наличие через `selOnlStck`.
7. Если официальная поверхность не отдаёт расположение товара внутри магазина, отвечать только про остатки.

## Пример

```js
const { lookupStoreProductAvailability } = require("daiso-product-search")

async function main() {
  const result = await lookupStoreProductAvailability({
    storeQuery: "강남역2호점",
    productQuery: "VT 리들샷 100"
  })

  console.log({
    store: result.selectedStore,
    product: result.selectedProduct,
    pickupStock: result.pickupStock,
    onlineStock: result.onlineStock
  })
}

main().catch((error) => {
  console.error(error)
  process.exitCode = 1
})
```

## Практические советы

- Если магазинов-кандидатов несколько, безопаснее показать верхние 2-3 и попросить уточнение.
- Если похожих товаров много, лучше показывать бренд, объём и вариант.
- Остатки не стоит трактовать как 100% гарантированное наличие в момент визита; полезно добавлять рекомендацию перепроверить перед выездом.
- Если официальная поверхность не знает полку или отдел, так и нужно писать: доступна только информация об остатке.

## Заметка по live-проверке

На 2026-03-27 следующие официальные вызовы возвращали реальные данные:

- `POST /api/ms/msg/selStr` -> кандидаты для `강남역2호점`
- `GET /ssn/search/SearchGoods?searchTerm=리들샷...` -> товарные кандидаты, включая `1049275`
- `POST /api/pd/pdh/selStrPkupStck` -> остаток для комбинации `strCd=10224`, `pdNo=1049275`

В том же smoke test комбинация `강남역2호점 + VT 리들샷 100` вернула остаток `0`. То есть официальный путь рабочий, но на тот момент в магазине товара не было.
