# Политика секретов

`ru-skill` объявляет только имена нужных переменных окружения. Способ поставки самих значений остаётся за агентом и пользовательской средой.

## Порядок разрешения credential

Все навыки, которым нужны секреты, используют один и тот же приоритет.

1. Если значение уже задано в окружении, оно используется напрямую.
2. Если агент умеет работать со своим secret vault, например 1Password CLI, Bitwarden CLI или macOS Keychain, он может взять значение оттуда и передать в окружение.
3. Если vault нет, сначала используется `~/.config/ru-skill/secrets.env`, а при его отсутствии допускается legacy fallback `~/.config/k-skill/secrets.env`; оба файла должны быть в формате dotenv с правами `0600`.
4. Если секрета нет ни в одном месте, агент должен запросить его у пользователя и сохранить через вариант 2 или 3.

Хранение в стандартном пути является fallback-механизмом, а не обязательным требованием.

## Стандартный secrets-файл

- Предпочтительный путь: `~/.config/ru-skill/secrets.env`
- Legacy fallback: `~/.config/k-skill/secrets.env`
- Явное переопределение: `RU_SKILL_SECRETS_FILE`, затем `KSKILL_SECRETS_FILE`
- Формат: plain dotenv (`KEY=value`, одна переменная на строку)
- Права: `0600` (только владелец может читать и писать)

```dotenv
KSKILL_SRT_ID=replace-me
KSKILL_SRT_PASSWORD=replace-me
KSKILL_KTX_ID=replace-me
KSKILL_KTX_PASSWORD=replace-me
SEOUL_OPEN_API_KEY=replace-me
AIR_KOREA_OPEN_API_KEY=replace-me
KSKILL_PROXY_BASE_URL=https://k-skill-proxy.nomadamas.org
```

## Политика при отсутствии секрета

Если у навыка с авторизацией не хватает значения, агент не должен обходить это сторонними сервисами или неофициальными трюками.

- Пользователю нужно точно назвать отсутствующую переменную окружения.
- Получение значения должно идти по порядку разрешения credential.
- Нельзя автоматически подставлять альтернативные сайты, неофициальные API или хардкод.
- Отсутствие секрета не является причиной молча переключаться на другой сервис.

## Запрещённые практики

- Хранить в git файл с настоящими секретами.
- Писать в документации реальные пароли даже как пример.
- Автоматически включать неофициальные обходы только потому, что секрета нет.

## Упрощённая модель угроз

- `~/.config/ru-skill/secrets.env` остаётся обычным dotenv-файлом.
- Legacy-файл `~/.config/k-skill/secrets.env` пока тоже считается допустимым, потому что нужен для совместимости.
- Права `0600` защищают эти файлы от других пользователей той же машины.
- `.gitignore` нужен, чтобы файл не утёк в git.
- Агент может читать и писать этот файл, и это ожидаемое поведение.
- В среде OpenClaw/агентов предполагается, что пользователь осознанно делегирует агенту работу с credential.

## Стандартные имена переменных

- `KSKILL_SRT_ID`
- `KSKILL_SRT_PASSWORD`
- `KSKILL_KTX_ID`
- `KSKILL_KTX_PASSWORD`
- `SEOUL_OPEN_API_KEY`
- `AIR_KOREA_OPEN_API_KEY`
- `KSKILL_PROXY_BASE_URL`

Все навыки этого репозитория, которым нужны credential, опираются на эту политику. Общий порядок настройки описан в [гайде по setup](setup.md).
