# Roadmap

## 현재 단계

`ru-skill`의 단기 목표는 "기존 `k-skill` 자산을 그대로 끌고 가는 저장소"에서 "러시아 및 러시아어 사용자 기준의 실사용 스킬 저장소"로 바꾸는 것입니다.

우선순위는 다음 순서로 둡니다.

1. 저장소 브랜드, 문서, 릴리스 메타데이터 정리
2. 공개 API 또는 공식 웹 표면이 있는 러시아/국제 공용 시나리오 추가
3. 기존 한국 전용 기능은 레거시로 유지하거나 별도 저장소로 분리 검토

## 기존 출시 기능

- 우편번호 검색
- K리그 경기 결과 조회 스킬 출시
- 택배 배송조회 스킬 출시
- 토스증권 조회 스킬 출시
- 사용자 위치 미세먼지 조회 스킬 출시
- 근처 블루리본 맛집 스킬 출시
- 근처 술집 조회 스킬 출시
- 다이소 상품 조회 스킬 출시

## 우선 전환 대상

### 생활 정보

#### 러시아 우편번호 / 주소 정규화

- 장점: 비교적 공개 표면이 많고, 다른 배송/주소 스킬의 기반이 된다
- 이유: 주소 체계 대응은 지역 생활형 스킬의 출발점이다

#### 날씨 / 대기질 / 경보

- 장점: 공개 데이터 소스와 프록시 패턴을 재사용하기 쉽다
- 이유: 도시별 조회 수요가 크고 러시아어 응답 정규화 가치가 높다

#### 대중교통 / 장거리 열차 조회

- 장점: 시각표와 역명 정규화가 되면 반복 사용성이 높다
- 이유: 기존 철도 예약 관련 구조를 재사용하기 좋다

### 전자상거래와 가격 확인

#### 마켓플레이스 상품 검색과 재고 확인

- 장점: 사용자 가치가 즉각적이고 테스트 시나리오를 만들기 쉽다
- 이유: 기존 `daiso-product-search` 같은 흐름을 러시아 서비스로 바꾸기 좋다

#### 근처 매장 / 장소 검색

- 장점: 위치 질의, 필터링, 영업 상태 요약 패턴을 재사용할 수 있다
- 이유: 기존 `blue-ribbon-nearby`, `kakao-bar-nearby`의 UX를 러시아 현실에 맞게 치환하기 좋다

### 금융과 계정성 워크플로

#### 증권/계좌 읽기 전용 요약

- 장점: 읽기 전용 래퍼는 위험을 낮추면서 실사용성을 확보하기 쉽다
- 이유: 기존 `toss-securities` 패턴을 러시아어 워크플로에 맞게 일반화할 수 있다

#### 세금 / 공공 민원 가이드형 스킬

- 장점: 완전 자동화가 아니어도 문서화와 단계별 지원만으로 가치가 있다
- 이유: 국가별 차이가 큰 영역이라 러시아어 현지화의 효과가 크다

## 유지 또는 정리 대상

### 유지

- `k-skill-proxy`: 무료 API용 프록시 베이스로 유지
- `hwp` 관련 문서: 러시아 전환과 별개로 문서 처리 유틸리티로 의미가 있음

### 레거시

- `k-lotto`
- `kleague-results`
- `daiso-product-search`
- `blue-ribbon-nearby`
- `kakao-bar-nearby`
- `toss-securities`

이 패키지들은 당장 삭제하지 않지만, 루트 저장소의 주 메시지는 러시아/러시아어 전환에 맞게 계속 정리합니다.
