# Результаты K League

## Что делает этот навык

Запрашивает результаты матчей K League1/K League2 за определённую дату через официальный JSON K League и при необходимости фильтрует по конкретной команде (например: `FC서울`, `서울 이랜드`, код команды `K09`), затем показывает текущую турнирную таблицу.

## Когда использовать

- "Покажи результаты K League1 за сегодня"
- "Покажи результаты FC서울 за 2026-03-22 и текущую турнирную таблицу"
- "Подведи итоги K League2 за 2026-03-22"

## Предварительные требования

- Node.js 18+
- `npm install -g kleague-results`

## Входные данные

- Дата: `YYYY-MM-DD`
- Лига: `K리그1` или `K리그2` (по умолчанию `K리그1`)
- Опционально: название команды, полное имя, код команды (`서울`, `FC서울`, `K09` и т.д.)

## Рабочий процесс

### 0. Установка пакета при отсутствии

Если `kleague-results` отсутствует в `npm root -g`, сначала попробуйте глобальную установку Node-пакета вместо обходного HTML-парсинга.

```bash
npm install -g kleague-results
```

### 1. Получение официального JSON K League

Этот навык использует следующие официальные JSON endpoints вместо HTML-парсинга.

- Расписание/результаты: `https://www.kleague.com/getScheduleList.do`
- Турнирная таблица команды: `https://www.kleague.com/record/teamRank.do`

```bash
GLOBAL_NPM_ROOT="$(npm root -g)" node --input-type=module - <<'JS'
import path from "node:path";
import { pathToFileURL } from "node:url";

const entry = pathToFileURL(
  path.join(process.env.GLOBAL_NPM_ROOT, "kleague-results", "src", "index.js"),
).href;
const { getKLeagueSummary } = await import(entry);

const summary = await getKLeagueSummary("2026-03-22", {
  leagueId: "K리그1",
  team: "FC서울",
  includeStandings: true,
});

console.log(JSON.stringify(summary, null, 2));
JS
```

### 2. Адаптация для человека

Не выводите исходный JSON напрямую, обработайте по следующим критериям.

- Домашняя команда vs гостевая команда
- Время матча / статус завершения
- Счёт
- Текущая турнирная таблица
- Если запрошена конкретная команда, отфильтруйте только матчи этой команды

### 3. Компактный ответ

Если запрошена таблица результатов, сначала дайте краткое резюме по матчу. Если запрошена конкретная команда, сначала покажите только матчи этой команды и текущую турнирную таблицу.

## Считается выполненным, когда

- Есть резюме матчей по дате
- При запросе команды отфильтрованы только матчи этой команды
- Текущая турнирная таблица также представлена

## Режимы сбоев

- Если сайт K League изменит структуру ответов `getScheduleList.do` или `teamRank.do`, потребуется корректировка пакета
- Для дат до матча может返回 статус ожидания вместо результатов
- При коротких названиях типа `서울` может потребоваться различение `FC서울` / `서울 이랜드` в зависимости от лиги

## Примечания

- Этот навык только для запросов
- Запросы пользователя "сегодня/вчера" всегда преобразуйте в абсолютную дату (`YYYY-MM-DD`) перед выполнением
- При добавлении нового пакета после слияния требуется npm publish (на базе Changesets) для работы глобальной установки
- Подробные примеры использования следуют `docs/features/kleague-results.md` и `packages/kleague-results/README.md`
