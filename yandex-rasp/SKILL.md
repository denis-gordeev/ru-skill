## Что делает этот навык

Ищет станции, получает расписание рейсов для конкретной станции и ищет маршруты между двумя городами или станциями через API Яндекс.Расписаний. Охватывает электрички, поезда дальнего следования, автобусы и авиарейсы по России.

## Когда использовать

- Когда нужно узнать расписание электричек, поездов или автобусов для российской станции
- Когда нужно найти маршрут между двумя городами с пересадками
- Когда нужен lookup станции по названию

## Предварительные требования

- API-ключ Яндекс.Расписаний (бесплатный, без регистрации на сайте)
- `npm install yandex-rasp`
- Переменная окружения `YANDEX_RASP_API_KEY` или передача ключа через `opts.apiKey`

## Входы

- Поисковый запрос станции (название, например «Казанский вокзал»)
- Код станции Яндекс (например `s9600013`) для расписания
- Коды двух станций/городов для поиска маршрута
- Необязательные фильтры: дата, тип транспорта, направление

## Основной сценарий

1. Найти станции по названию через `searchStations`
2. Получить расписание для нужной станции через `getStationSchedule` или найти маршрут через `searchTrips`
3. Вернуть нормализованные результаты: время отправления/прибытия, номер рейса, перевозчик, платформа

## Готово, когда

- Найдены станции по названию с кодами Яндекс и ЕССР
- Возвращено расписание с временем отправления/прибытия, номером рейса и перевозчиком
- Возвращены маршруты между станциями с длительностью, ценой и наличием пересадок
- Ошибки API проброшены с понятным сообщением

## Режимы отказа

- Нет API-ключа — выбрасывается ошибка с инструкцией получить ключ
- Станция не найдена — пустой массив результатов
- API вернул ошибку — пробрасывается с кодом статуса и телом ответа

## Примечания

- Эндпоинт `stations_list` возвращает полный справочник всех станций (~40 МБ), имеет смысл кэшировать
- Клиент использует актуальный хост `api.rasp.yandex-net.ru`, а не устаревший `api.rasp.yandex.net`
- Read-only, без покупки билетов и write-операций
